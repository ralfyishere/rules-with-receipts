## The fix

```python
def take(n, items):
    """Return the first n items of the list."""
    return items[:n]
```

The original customer bug was a classic off-by-one. `items[:n + 1]` slices up to index `n+1`, so it returns **n+1 items instead of n**. The correct slice is `items[:n]`, since Python's upper bound is already exclusive.

- It drops the first element instead of including it (`take(1, ["a","b"])` → `["b"]` instead of `["a"]`).
- The length happens to come out right (`n+1 - 1 = n`), which is probably why it looked plausible — but every returned window is shifted one slot too far into the list.
