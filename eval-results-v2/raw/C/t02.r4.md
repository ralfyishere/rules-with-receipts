## What was actually happening

- `configparser` reads all values as **strings**, so `cfg["retries"]` was `"3"`, not `3`.
- `range("3")` raised `TypeError: 'str' object cannot be interpreted as an integer` on the *first* line of the loop — `fetch()` was never called.
- The bare `except Exception` caught that `TypeError` and unconditionally printed `"ERROR: network timeout - retrying later"`. The message is hardcoded and never inspects the real exception, so it slandered the network on every run.

**Fix:** `range(int(cfg["retries"]))` — one line, coercing the config string to an int.
