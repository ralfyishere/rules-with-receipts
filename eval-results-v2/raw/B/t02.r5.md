**What happened**

- `range("3")` throws `TypeError` before `fetch()` is ever called.
- The catch-all `except Exception` reported that type error as `"ERROR: network timeout - retrying later"` — a hardcoded message unrelated to the actual failure. So every run "failed" with a network error while the network was never contacted.

**Fix:** `range(cfg.getint("retries"))` — one line, uses configparser's built-in int coercion.

One thing worth flagging (I didn't change it, since you asked for the real cause): that `except Exception: print("network timeout")` block is a landmine. It will mislabel *any* future error — bad config keys, a real bug in `fetch`, whatever — as a network timeout.
